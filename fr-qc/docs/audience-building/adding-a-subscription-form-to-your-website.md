# Ajouter un formulaire d'abonnement à ton site web

<iframe width="560" height="315" src="https://www.youtube-nocookie.com/embed/QEhJ3QzE_Xo" frameborder="0" 
 allowfullscreen></iframe>
 
## Description

Les formulaires d'abonnement peuvent être inclus dans ton site web ou blogue pour que tes visiteurs puissent fournir leurs données et recevoir des courriels sur des sujets pertinents et/ou d'intérêt. Fondamentalement, un formulaire d'abonnement, c'est conçu pour enregistrer des abonnés à ta liste de diffusion - comme un carnet d'adresses moderne, mais en bien plus efficace !

## Pourquoi c'est important

Les formulaires d'abonnement, c'est un moyen légal et pratique pour les abonnés de te fournir leurs informations. Le processus de création et de configuration est vraiment simple - plus simple que de commander une poutine au restaurant ! Un des grands avantages du marketing par courriel, c'est qu'il te permet de cibler des groupes basés sur divers critères. En ajoutant certains de ces critères à ton formulaire d'abonnement, tu peux personnaliser tes courriels comme si tu connaissais tes clients depuis toujours. Comme règle générale cependant, demander trop d'informations peut mener à moins d'intérêt pour s'abonner - personne aime remplir un formulaire plus long qu'une déclaration d'impôts ! Tu vas devoir commencer par créer un formulaire qui est adapté à tes besoins.

## Instructions étape par étape

En utilisant notre système, tu peux intégrer un formulaire d'acceptation pour une liste de contacts spécifique directement dans ton site web - aussi facilement qu'ajouter une photo sur Facebook !

### Obtenir le formulaire pour ta liste

<Stepper>

1. Sélectionne l'onglet **Formulaires**
   ![Navigation de l'onglet Formulaires](/images/forms/DE2647B1-84B2-48DC-8709-CB03D406C3CC_4_5005_c.jpeg)

1. Clique sur **'...'** en haut à droite du formulaire d'abonnement puis sélectionne **Modifier**
   ![Menu d'options de formulaire avec bouton Modifier](/images/forms/6EE9D232-BC7A-419B-AD95-FC24288C49E0_4_5005_c.jpeg)

1. Tu vas avoir l'option de :
   - utiliser un lien direct vers le formulaire
   - récupérer le code HTML pour intégrer sur ton site
</Stepper>

### Formulaire lié

Il va te donner le lien direct vers le formulaire en cliquant sur **Obtenir l'URL**.

![Option Obtenir l'URL](/images/forms/994FE0B7-094B-41B1-8786-97C21DB033D8_4_5005_c.jpeg)

![Interface de l'éditeur de formulaire](/images/forms/D4898F41-2133-4A78-9EDA-E86A0CA5F44F_4_5005_c.jpeg)

Tu peux personnaliser ton formulaire en cliquant sur **Modifier le formulaire**.

![Personnalisation de modification de formulaire](/images/forms/89BDF6DB-831B-40A7-AEBA-2B1F77B4FDF1_4_5005_c.jpeg)

### Formulaire intégré

Il va te donner le code à intégrer sur ton site web en sélectionnant **Obtenir le code**.

![Option Obtenir le code](/images/forms/FE932224-C921-4065-84CB-582AADD2AB5C_4_5005_c.jpeg)

![Aperçu du formulaire intégré](/images/forms/FB295771-7AEB-4F17-9E75-B0C14F2AB255_4_5005_c.jpeg)

N'hésite pas à personnaliser ton formulaire en cliquant sur **Modifier le formulaire**.

![Interface de personnalisation de formulaire](/images/forms/87056EFD-3136-4C7E-9F20-88E977D2D298_4_5005_c.jpeg)

### Modifier ton formulaire

Les éléments de formulaire suivants sont disponibles dans ton éditeur :

- **Titre** avec six tailles H différentes
- **Paragraphe :** Décris le type de communication qui va être envoyé. Communication sur une base hebdomadaire ou mensuelle, par exemple. Codes d'inscription, codes promo, etc.
- **Attribut personnalisé :** Un attribut qui peut être personnalisé
- **Courriel** (obligatoire et par défaut)
- **Bouton d'abonnement** (obligatoire et par défaut)
- **Attribut caché** dans les cas où tu veux qu'un segment soit populé par défaut

![Personnalisation d'élément de formulaire](/images/forms/50A54B79-A700-4E53-83CA-E43A179A3378_4_5005_c.jpeg)

Tu peux sélectionner ces éléments pour les personnaliser. C'est important d'éviter de demander les mauvaises informations : tu veux pas que tes clients trouvent ça overwhelming, alors ils cessent simplement de s'abonner. Trois ou quatre champs devraient suffire.

![Options d'élément de formulaire](/images/forms/1F845091-4D60-4758-A9BD-DE622A2E6E01_4_5005_c.jpeg)

- Clique sur l'**icône de crayon** pour modifier un élément
- Clique sur le **double carré avec un signe plus** pour dupliquer un élément
- Clique sur l'**icône de poubelle** pour supprimer un élément

## Ton formulaire d'abonnement est prêt !

Utilise-le pour bâtir ta précieuse liste et continue à générer des prospects pour la croissance de ton compte. Les données que tu assembles vont fournir une source de nouveaux prospects pour ton organisation, lui permettant de continuer à s'étendre. De plus, si tu as des champs personnalisés sur ta liste, ils vont être ajoutés au formulaire.

## Dépannage

**Le formulaire ne s'affiche pas sur mon site web**

- Vérifie que le code d'intégration a été copié complètement sans caractères manquants
- Check si les paramètres de sécurité de ton site web bloquent les scripts externes
- Assure-toi que le code de formulaire est placé dans les balises body de ton HTML
- Teste dans différents navigateurs pour éliminer les problèmes spécifiques au navigateur

**Les soumissions n'apparaissent pas dans ma liste**

- Confirme que le formulaire est connecté à la bonne liste de contacts
- Vérifie si le double opt-in est activé (les abonnés doivent confirmer)
- Check que ta liste n'a pas atteint sa limite d'abonnés
- Recherche les soumissions dans le statut en attente ou non confirmé

**Le style du formulaire ne match pas avec mon site web**

- Utilise les options de personnalisation CSS dans l'éditeur de formulaire
- Ajoute du CSS personnalisé pour remplacer les styles par défaut
- Vérifie si le CSS de ton site web entre en conflit avec les styles de formulaire
- Considère utiliser l'option d'intégration iframe pour une meilleure isolation

**Messages d'erreur lors de la soumission du formulaire**

- Assure-toi que tous les champs requis sont remplis correctement
- Vérifie si la validation de courriel bloque certains formats de courriel
- Check que le formulaire n'a pas été modifié après l'intégration
- Teste avec différentes adresses courriel pour identifier des patterns
