## Support for more webhook events

### Adding more webhooks

More events can trigger webhooks. Indeed, you can define callbacks when audience resources (lists, segments and contacts) are added, removed or updated. You can also define triggering webhooks upon campaign delivery or actions, such as when subscribers open emails and click a link in emails sent from automation, campaigns or transactional emails.

[How to create and manage Webhooks](https://cakemail.dev/v1.6/docs/using-webhooks)  
[Webhook API reference](https://cakemail.dev/v1.6/reference#webhooks)

### Usage of role-based email addresses

It is now possible to use role-based email addresses as a user or sender. A role-based email address is usually a corporate address that is not associated with a single, specific person, such as sales@ info@ or team@ email addresses.

[Role-based emails](https://support.cakemail.com/hc/en-us/articles/360056244474-Role-based-emails).
